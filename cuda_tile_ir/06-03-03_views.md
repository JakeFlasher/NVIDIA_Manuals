### [6.3.3. Views](https://docs.nvidia.com/cuda/tile-ir/latest/sections#views)[](https://docs.nvidia.com/cuda/tile-ir/latest/sections/#views "Permalink to this headline")

**Tile IR** provides a set of view types as described in [Tensor View](https://docs.nvidia.com/cuda/tile-ir/latest/sections/types.html#type-views).

Views provide a structured views of memory by enriching a pointer with additional data.
Views have their own set of memory operations [cuda_tile.load_view_tko](https://docs.nvidia.com/cuda/tile-ir/latest/sections/operations.html#op-cuda-tile-load-view-tko) and [cuda_tile.store_view_tko](https://docs.nvidia.com/cuda/tile-ir/latest/sections/operations.html#op-cuda-tile-store-view-tko)
which vary in behavior based on the view type and make use of the additional metadata. Due to the face that views are not a
singular type, but a family of types each concrete view type has its own value representation.

Our first-order view type is [Tensor View](https://docs.nvidia.com/cuda/tile-ir/latest/sections/types.html#type-tensor-view). A tensor view value is logically a tuple of \(\((ptr, \vec{shape}, \vec{strides}, \vec{dimgroups})\)\), and load and store operate
on the entire tensor view. The index space of this view is rank-0 (i.e., there is only a single element in the index space).

Our primary second-order (or “sub-view”) view type is [Partition View](https://docs.nvidia.com/cuda/tile-ir/latest/sections/types.html#type-partition-view).

The partition view type [Partition View](https://docs.nvidia.com/cuda/tile-ir/latest/sections/types.html#type-partition-view) is a subview type which represents a tensor view partitioned into tiles.
A partition view value is logically a tuple of \(\((tensor\_view, \vec{tile\_size})\)\), where load and store operate over
tile values of the given size. The index space of this view is:

$$
\[indexSpace[i] = \text{tensorViewShape}[\text{dimGroups}[i]] / \text{tileSize}[i]\]
$$

For example a tensor view with shape \(\((1024, 1024)\)\) may be partitioned into a grid of \(\(64 \times 32\)\) tiles,
resulting in a partition view with shape \(\((16, 32)\)\). See [Type System](https://docs.nvidia.com/cuda/tile-ir/latest/sections/types.html#section-types) for more details on the different types of views.

> **Note**
>
> As with tiles, the physical layout and memory representation of a view is not visible to the program and is
> not specified by the language semantics.

> **Warning**
>
> Reading or writing of bounds of any allocation is undefined behavior and bounds checking must be performed by the programmer
> if desired.
